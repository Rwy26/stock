## 1. 시스템 개요

Apollo는 한국투자증권(KIS) OpenAPI를 활용한 **AI 기반 주식 자동매매 플랫폼**입니다.

### 핵심 기능 요약

| 기능 | 설명 |
|------|------|
| 대시보드 | 실시간 계좌 현황, 시장 지수, 환율/금리 차트 |
| 추천 종목 | 14개 지표 기반 AI 종목 추천 (최대 100점) |
| SA 자동매매 | 골든크로스 예측 기반 스마트 자동매매 |
| Plus 자동매매 | 포트폴리오 순환 매매 (언더퍼포머 자동 교체) |
| SV Agent | 멀티 AI 에이전트 감독 자동매매 |
| 관리자 | 사용자 관리, 로그인 이력 |

### 기술 스택

- **백엔드**: Python 3.10+ / FastAPI / MySQL 8.0
- **프론트엔드**: React 18 / TypeScript / Material UI
- **인증**: JWT 토큰
- **증권 연동**: 한국투자증권 KIS OpenAPI

## 2. 시작하기

### 2-1. 접속 방법

```
URL: https://songstock.art
백엔드 포트: 5001
프론트엔드 포트: 3001
```

### 2-2. 로그인

1. 브라우저에서 서비스 URL로 접속합니다.
2. **이메일 주소**와 **비밀번호**를 입력합니다.
3. **로그인** 버튼을 클릭합니다.
4. 로그인 성공 시 대시보드로 이동합니다.

> **기본 계정**  
> - 관리자: `administrator` / `ChangeMe!` (또는 `backend/.env`의 `ADMIN_PASSWORD`)  
> - 일반 사용자: 관리자에게 발급 요청

### 2-3. 프로필 설정 (최초 1회)

최초 로그인 후 **프로필 설정** 화면이 표시됩니다.

1. **닉네임** 입력
2. **KIS API 정보** 입력
   - KIS 앱키 (App Key)
   - KIS 앱 시크릿 (App Secret)
   - 계좌번호 (앞 8자리)
   - 거래 구분: 실계좌 / 모의투자
3. **저장** 버튼 클릭

> ⚠️ KIS API 정보는 한국투자증권 홈페이지에서 발급받아야 합니다.  
> KIS 웹 → MY → 고객서비스 → Open API → 서비스 신청

## 3. 대시보드 (홈)

홈 화면은 **3×3 그리드** 레이아웃으로 핵심 정보를 한눈에 보여줍니다.

### 3-1. 자산 요약 (Asset Summary)

- **총 평가금액**: 보유 주식 + 예수금 합계
- **총 투자금액**: 매수에 투입된 금액
- **수익금**: 현재 평가손익
- **수익률 (%)**: 투자 대비 수익 비율
- **예수금**: 현재 사용 가능한 현금

### 3-2. 자산 추이 차트

- 최근 30일간 **일별 총 자산 변화** 선 그래프
- 날짜별 hover 시 상세 금액 표시

### 3-3. 자동매매 상태

- SA / Plus / SV Agent의 **활성화 여부** 상태 표시
- 각 엔진별 오늘 거래 건수 표시

### 3-4. Top 추천 종목

- 오늘 날짜 기준 **상위 5개 추천 종목** 빠른 조회
- 점수, 현재가, 등락률 표시

### 3-5. 일봉 캔들 차트

- 선택 종목의 **일봉 OHLCV** 차트
- 기술적 분석 지원

### 3-6. 시장 지수 차트

- **KOSPI / KOSDAQ** 당일 분봉 차트
- 시장 흐름 실시간 모니터링

### 3-7. 금리 차트

- 한국 기준금리 및 CD금리 추이

### 3-8. 환율 차트

- USD/KRW 환율 추이 (최근 30일)

### 3-9. 보유 종목 (포트폴리오 요약)

- 계좌 기준 보유 종목을 표로 요약 표시
- 종목/수량/현재가/수익률을 빠르게 확인

## 4. 포트폴리오

### 4-1. 포트폴리오 화면 구성

| 항목 | 설명 |
|------|------|
| 종목명 | 보유 종목 |
| 수량 | 보유 주식 수량 |
| 평균 매수가 | 평균 매입 단가 |
| 현재가 | KIS API 실시간 조회 가격 |
| 평가금액 | 현재가 × 수량 |
| 손익금 | 평가금액 - 투자금액 |
| 수익률 | 손익 / 투자금 × 100 |
| 매수일 | 최초 또는 마지막 매수일 |

### 4-2. 포트폴리오 동기화

- **KIS 실시간 동기화**: 한국투자증권 실제 계좌와 자동 동기화
- 포트폴리오 화면 새로고침 시 최신 데이터 반영

> ⚠️ **현재가 주의**: 현재가는 반드시 KIS 실시간 API에서 조회합니다.  
> DB의 daily_prices 값으로 대체하지 않습니다 (오류 방지).

## 5. 종목 탐색

### 5-1. 종목 검색

- 상단 검색창에 **종목명** 또는 **종목코드** 입력
- 실시간 자동완성 제공
- 종목 클릭 시 상세 화면으로 이동

### 5-2. 종목 상세 (Stock Detail)

- **현재가 / 등락률**
- **일봉 차트** (캔들스틱)
- **14개 지표 점수** 상세 표시
  - 가치지표: PER, PCR, PBR, 부채비율 (합계 29점)
  - 수급지표: 외국인순매수, 기관순매수, 거래대금 (합계 26점)
  - 수익지표: ROE, 영업이익률, 매출증가율 (합계 21점)
  - 성장지표: 이익증가율 (합계 6점)
  - 기술지표: OBV, MFI, 매집패턴 (합계 18점)
- **관심 종목 추가** 버튼

## 8. 일반 자동매매

### 8-1. 개요

기본 자동매매 엔진으로, 추천 점수 기반의 **단순 매수/매도 자동화** 기능입니다.

### 8-2. 설정 항목

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 활성화 여부 | On/Off 토글 | Off |
| 총 예산 | 자동매매에 사용할 총 금액 | - |
| 종목당 예산 | 종목 1개에 투자할 금액 | - |
| 최대 보유 종목 수 | 동시 보유 가능한 최대 종목 수 | 5 |
| 손절률 | 이 수익률 이하 시 자동 손절 | -5% |
| 익절률 | 이 수익률 이상 시 자동 익절 | +10% |

### 8-3. 운영 시간

- **거래일 09:00 ~ 15:20** 자동 실행 (1분 주기)
- 주말 / 공휴일 자동 스킵

## 9. SA 자동매매 (Smart Agent)

### 9-1. 개요

SA(Smart Agent) 자동매매는 **골든크로스 예측 + OBV/MFI/매집패턴 가중치** 기반의 고도화된 자동매매 시스템입니다.

### 9-2. 매수 조건

매수는 다음 조건이 **모두 충족**될 때 실행됩니다:

1. **추천 점수** ≥ 설정된 최소 점수 (default: 50점)
2. **골든크로스 예측**: 5일선이 20일선 상향 돌파 임박
3. **거래량 급증**: 최근 거래량이 평균 대비 1.5배 이상
4. **이격도 조건**: 현재가 ≤ 이동평균 × 98% (과열 종목 제외)
5. **기존 포트폴리오 미보유** (SA가 매수한 종목만 관리)

### 9-3. 매도 전략

#### 손절
- 기본 손절: **-5.0%** 이하 시 자동 매도
- 골든크로스 달성 후 손절: **-2.0%** (추세 초입 보호)

#### 분할 익절
- **1차 익절 (+5%)**: 보유 수량의 50% 매도
- **2차 익절 (+10%)**: 나머지 전량 매도

#### 트레일링 스탑 (최고가 대비 하락 시 매도)

| 수익 구간 | 트레일링 낙폭 |
|-----------|--------------|
| +3% ~ +5% | 고점 대비 -1.5% |
| +5% ~ +10% | 고점 대비 -1.8% |
| +10% 이상 | 고점 대비 -2.5% |

#### 금요일 청산 (주말 리스크 회피)
- **금요일 15:15** 이후 보유 전 종목 자동 청산

#### 시장 마감 청산
- **매일 15:20** 잔여 포지션 정리

### 9-4. 설정 항목

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 상태 | On/Off | Off |
| 총 예산 | 자동매매 전용 총 예산 | - |
| 종목당 예산 | 종목 1개 투자 금액 | - |
| 최대 종목 수 | 동시 보유 가능 종목 수 | 5 |
| 최소 SA 점수 | 매수 대상 최소 점수 | 50 |
| 주의 모드 최소 점수 | 시장 주의 시 최소 점수 | 55 |
| 탐색 순위 범위 | 추천 종목 상위 몇 개까지 탐색 | 200 |
| OBV 가중치 | OBV 지표 가중치 (0.5~1.5) | 1.0 |
| MFI 가중치 | MFI 지표 가중치 (0.5~1.5) | 1.0 |
| 패턴 가중치 | 매집패턴 가중치 (0.5~1.5) | 1.0 |
| 최대 보유일 | 이 일수 초과 시 강제 매도 | 10일 |
| 변동성 조정 | 고변동성 종목 필터링 | On |
| 모니터링 주기 | 가격 체크 간격 | 1분 |

### 9-5. 화면 구성

#### 탭 1: 현황
- 현재 보유 포지션 (종목명, 수량, 매수가, 현재가, 수익률)
- 골든크로스 달성 여부

#### 탭 2: 매매 로그
- 매수/매도 이력, 이유, 손익 기록

#### 탭 3: 알림
- 중요 이벤트 알림 (매수 실패, 손절 발생, 예산 부족 등)
- 읽음/안읽음 표시

#### 탭 4: 시그널
- 매수/매도 시그널 발생 내역
- 실행 여부 및 실패 사유

#### 탭 5: 일별 성과
- 날짜별 수익금/수익률 차트
- 누적 손익 추이

#### 탭 6: 계좌 스냅샷
- 시간대별 계좌 현황 기록

### 9-6. 실현손익 조회

- **전체 기간** 실현손익 합계
- **월별** 수익률
- **연간** 수익률
- 승/패 횟수 통계

## 10. Plus 자동매매

### 10-1. 개요

Plus 자동매매는 **포트폴리오 순환 매매** 시스템입니다.  
최대 N개 종목을 유지하면서, 성과가 낮은 종목을 더 좋은 종목으로 자동 교체합니다.

### 10-2. 핵심 로직

```
1. 추천 종목 랭킹에서 매수 대상 탐색
2. 매수 → 포지션 유지
3. 손절/익절/트레일링 스탑 조건 확인
4. Underperformer 감지 시 → 매도 후 상위 랭크 종목으로 교체
```

### 10-3. 매수 조건

- **추천 랭킹** 범위 내 종목 (search_rank_start ~ search_rank_end)
- 상승 요구 옵션 활성화 시: 전일 대비 **상승 중인 종목** 우선
- 최소 SA 점수 조건 충족 종목만 편입
- 현재 보유 종목 수 < 최대 종목 수

### 10-4. 매도 조건

| 매도 유형 | 조건 |
|-----------|------|
| 손절 (Stop Loss) | 수익률 ≤ 설정 손절률 |
| 익절 (Take Profit) | 수익률 ≥ 설정 익절률 |
| 트레일링 스탑 | 최고가 대비 설정률 하락 |
| 급락 청산 (Crash Sell) | 매수 후 급격한 하락 감지 |
| 포트폴리오 순환 | 하위 성과 종목 → 상위 랭크 종목으로 교체 |
| 최소 보유 시간 | 최소 보유 시간 미충족 시 매도 보류 |

## 11. SV Agent (AI 슈퍼바이저)

### 11-1. 개요

SV Agent는 **멀티 AI 에이전트 기반 감독 자동매매** 시스템입니다.  
AI가 매수/매도 신호를 생성하고, 규칙 기반 또는 AI 기반으로 최종 승인을 거쳐 주문을 집행합니다.

### 11-2. 운영 모드

| 모드 | 설명 |
|------|------|
| `rule_based` | 규칙 기반 자동 승인 (기본 설정 충족 시 바로 실행) |
| `ai_assisted` | AI 보조 모드 (AI 검토 후 규칙 기반 승인) |
| `full_ai` | 전체 AI 모드 (AI가 최종 승인 결정) |

### 11-3. AI 에이전트 구성

```
SV Agent 멀티 에이전트 구조

[데이터 수집 에이전트]
  └─ 매일 OHLCV, 재무지표, 뉴스 데이터 수집
  
[분석 에이전트]  
  └─ 14개 기술/기본 지표 계산
  
[추천 에이전트]
  └─ AI 점수 산출 (머신러닝/딥러닝 모델)
  
[리스크 에이전트]
  └─ 포트폴리오 리스크 평가
  
[검증 에이전트]
  └─ 매매 신호 검증
  
[슈퍼바이저 에이전트]
  └─ 최종 승인 및 집행 결정
```

### 11-4. AI 학습 시스템

- **일별 데이터 수집**: 매일 새벽 OHLCV + 재무 데이터 자동 수집
- **레이블링**: Triple Barrier Method로 매수/매도/보유 레이블 자동 생성
- **주간 재학습**: 매주 최신 데이터로 모델 재학습 (incremental learning)
- **모델 저장**: ONNX 형식으로 모델 최적화 저장

### 11-5. Kill Switch (긴급 정지)

킬 스위치 기준 손실률을 초과하면 **모든 포지션 즉시 청산** 후 비활성화.

| 설정 | 설명 | 기본값 |
|------|------|--------|
| Kill Switch 임계값 | 발동 기준 손실률 | -10% |
| 현재 손실률 | 현재 누적 손실률 (실시간) | - |

### 11-6. 설정 항목

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 활성화 | On/Off | Off |
| 총 예산 | SV Agent 전용 예산 | - |
| 최대 종목 수 | 동시 보유 가능 종목 수 | 5 |
| 종목당 최대 예산 | 종목당 투자 상한액 | - |
| 승인 모드 | rule_based / ai_assisted / full_ai | rule_based |
| 손절률 | 기본 손절 기준 | -5% |
| 익절률 | 기본 익절 기준 | +10% |
| 트레일링 시작 수익률 | 트레일링 스탑 시작 기준 | +3% |
| 최대 보유일 | 자동 강제 청산 일수 | 10일 |
| AI 학습 활성화 | 지속적 학습 여부 | On |
| Kill Switch 임계값 | 긴급 정지 손실률 | -10% |

### 11-7. 화면 구성

#### 탭 1: 보유 포지션
- SV Agent가 매수한 종목 목록
- 종목명, 수량, 매수가, 현재가, 수익률
- KIS 실시간 API로 현재가 조회

#### 탭 2: AI 추천
- AI가 생성한 매수 추천 종목
- AI 예측 점수, 추천 사유
- 승인 대기 건수

#### 탭 3: 승인 이력
- 승인/거절된 매매 신호 이력
- 승인 사유, 거절 사유 표시

#### 탭 4: 실현손익
- 청산된 포지션의 실현 손익
- 날짜별, 종목별 집계
- 승/패 통계

#### 탭 5: 설정
- SV Agent 전체 설정 변경
- Kill Switch 상태 실시간 표시

### 11-8. 실시간 모니터

백엔드에서 `SVAgentRealtimeMonitor`가 별도 스레드로 실행되며:
- 보유 포지션 현재가 실시간 추적
- 손절/익절 조건 자동 체크
- 알림 발송 (이메일)

## 12. 주말 매매 / SA 주말 청산

### 12-1. SA 금요일 자동 청산

SA 자동매매는 **금요일 15:15** 보유 전 종목을 자동 청산합니다.

- **목적**: 주말 간 예기치 못한 이벤트로 인한 리스크 회피
- **대상**: SA 자동매매로 매수된 전 종목
- **동작**: 15:15에 시장가 매도 주문 자동 발행

### 12-2. 주말 매매 설정 (SA Weekend Sell)

SA 주말 청산 설정 화면에서:

1. **금요일 청산 활성화** On/Off
2. **청산 시각** 설정 (기본: 15:15)
3. **대상 종목 범위** 설정

### 12-3. 월요일 재진입

월요일 장 시작 후 기존 전략 조건에 따라 **자동 재매수** 진행.

- 관리자 JWT 토큰은 **20시간 주기**로 자동 갱신
- 서버 재시작 없이 토큰 만료 방지

### 14-4. 자동매매 운영 안전장치 (Live Orders / Kill Switch)

자동매매는 운영 안전을 위해 **서버 환경 변수 기반 안전 게이트**를 사용합니다.

| 항목 | 기본값 | 설명 |
|------|--------|------|
| `AUTOTRADING_LIVE_ORDERS` | `0` | **실계좌 주문 허용 게이트**. `0`이면 실계좌 주문은 거부되며 엔진 로그에 오류로 남습니다. `1`로 설정 시에만 실계좌 주문이 허용됩니다. |
| `AUTOTRADING_KILL_SWITCH` | `0` | **긴급 정지 스위치**. `1`이면 스케줄러 기반 자동 tick이 중지됩니다(긴급 상황에서 즉시 사용). |

> 참고: 모의투자(페이퍼) 계정은 `AUTOTRADING_LIVE_ORDERS`의 영향을 받지 않습니다.

### 14-5. 엔진 점검용 수동 Tick (관리자)

운영 점검을 위해, 관리자는 특정 사용자에 대해 **1회성 수동 tick**을 실행할 수 있습니다.

- 목적: “설정 저장 → 1회 tick → 엔진 로그 확인”으로 설정 반영/엔진 동작을 빠르게 점검
- 안전 가드: `AUTOTRADING_KILL_SWITCH=1`이면 실행되지 않습니다
- API: `POST /api/admin/engine/tick-once`

### 14-6. 엔진 로그 확인 (관리자)

엔진의 `buy/sell/skip/error` 등 동작 결과는 엔진 로그에서 확인합니다.

- API: `GET /api/admin/engine-logs?userId=<id>&limit=...`

## 16. 자주 묻는 질문

### Q1. 자동매매가 실행되지 않습니다.

**확인 사항:**
1. 해당 자동매매 엔진이 **On** 상태인지 확인
2. KIS API 토큰이 유효한지 확인 (`/api/kis/token-status`)
3. 거래 가능 시간(09:00~15:20) 내인지 확인
4. 주말/공휴일 여부 확인
5. 예산 잔액이 충분한지 확인

### Q2. 추천 종목이 업데이트되지 않습니다.

**확인 사항:**
1. 야간 배치가 정상 완료되었는지 로그 확인
2. `indicator_scores` 테이블의 최신 날짜 확인
3. 수동 배치 실행: `python 1_backend/scripts/nightly_batch.py`

### Q3. 포트폴리오 현재가가 어제 종가로 표시됩니다.

- 거래 시간 외(장 마감 후)에는 마지막 거래가 표시가 정상입니다.
- 장 중임에도 어제 종가라면 KIS API 연결을 확인하세요.

### Q4. SA 자동매매가 금요일에 청산하지 않았습니다.

**확인 사항:**
1. `SAAutoTradingEngine`의 금요일 청산 기능이 활성화되어 있는지 확인
2. `FRIDAY_LIQUIDATION_TIME = "15:15"` 이후 실행 여부 확인
3. SA 자동매매 엔진 자체가 On 상태인지 확인

### Q5. SV Agent Kill Switch가 발동되었습니다.

1. 전체 포지션이 자동 청산됩니다.
2. SV Agent가 자동으로 **비활성화**됩니다.
3. 원인 분석 후 수동으로 킬 스위치 임계값을 조정하고 재활성화하세요.

### Q6. 배치 로그 확인 방법

```powershell

```
