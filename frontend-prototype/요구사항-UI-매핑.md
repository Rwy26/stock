# Apollo UI 매핑 (요구사항 기준)

본 문서는 요구사항 문서를 기준으로 재미나이 UI를 적용한 화면 매핑표입니다.

## 1. 공통 원칙

- 레이아웃: 좌측 메뉴 + 우측 콘텐츠 패널
- 톤앤매너: 글래스 카드, 강한 대비, 다층 그라디언트
- 핵심 지표: 상단 KPI 카드 우선 노출
- 실시간 상태: 엔진 On/Off 상태를 칩 형태로 강조
- 반응형: 1100px 이하 1열 전환, 720px 이하 모바일 최적화

## 2. 요구사항 대비 구현 항목

| 요구사항 메뉴 | 프로토타입 반영 위치 |
|---|---|
| 시작하기 - 로그인 | login.html |
| 시작하기 - 프로필 설정 | profile-setup.html |
| 대시보드 | index.html (자산요약/자산추이/엔진상태/Top추천/캔들/지수/금리/환율) |
| 포트폴리오 | portfolio.html (KIS 실시간 현재가 원칙 문구 포함) |
| 종목 탐색 | stock-search.html (검색 + 상세 + 지표 점수 + 관심 추가) |
| 추천 종목 | recommendations.html (필터/정렬 + 관심 추가) |
| 관심 종목 | watchlist.html (목록 + 삭제) |
| 일반 자동매매 | auto-basic.html (설정 + 운영시간 표기) |
| SA 자동매매 | auto-sa.html (설정 + 6개 탭: 현황/로그/알림/시그널/일별성과/스냅샷) |
| Plus 자동매매 | auto-plus.html (설정 + 4개 탭: 현황/로그/순환이력/성과차트) |
| SV Agent | sv-agent.html (설정 + Kill Switch + 5개 탭) |
| 관리자 기능 | admin.html (사용자 관리 + 로그인 이력) |

## 3. 다음 구현 단계

1. (완료) 메뉴별 실제 화면(HTML)로 분리
2. mock 데이터(JSON) 준비 후, 런타임 확보 시 API 연동으로 교체
3. 차트 영역을 실데이터 바인딩(자산/지수/금리/환율/캔들)
4. 자동매매 로그/알림/시그널/성과 탭을 백엔드 데이터로 치환
